## 目的与概述

**优惠券管理小程序 (Voucher Management Applet)** 是一款功能强大的促销和票务引擎，旨在简化现代营销运营。它允许营销和销售团队创建高度定制的折扣规则，生成唯一的二维码门票，并管理自动化的忠诚度计划——所有这些都无需开发人员的协助。

该小程序弥合了营销活动与销售点（POS）或电子商务结账之间的关键差距，提供实时验证和投资回报率（ROI）跟踪，以保护您的利润。

{{< callout type="info" >}}
**核心概念：活动 (Campaign) vs 优惠券 (Voucher) vs 门票 (Ticket)**
- **活动 (活动逻辑):** 总体逻辑。例如，“12 月所有鞋子打 9 折”。您在这里配置*规则*。
- **优惠券 (代码):** 客户在结账时输入或出示的实际字母数字字符串（例如 `DEC10`）。
- **门票 (凭证):** 用于物理或虚拟进入的专用优惠券（例如，在活动门前扫描的二维码）。
{{< /callout >}}

## 战略配置：构建您的优惠券逻辑

创建优惠券不仅仅是填写表单，更是设计一套财务规则，在提升客户体验的同时保护您的利润。

{{< callout type="tip" >}}
**创建清单：开始前的准备工作**
- [ ] **目标：** 您是需要现金抵扣 (结算券)、百分比折扣 (折扣券) 还是忠诚度积分 (奖励券)？
- [ ] **经济性：** 您愿意为这次客户行为支付的最高“价格”是多少？
- [ ] **安全性：** 该代码是否需要高度保护 (Softpin) 还是通用的公共代码？
- [ ] **时间表：** 活动何时开始 (领取窗口) 以及代码何时生效 (使用窗口)？
{{< /callout >}}

### 第 1 步：定义蓝图 (主要详细信息)

本部分定义了优惠券的“身份证”。请仔细选择设置，因为部分设置在活动上线后将无法更改。

| 设置 | 战略用例 | 操作要求 |
|:---:|---|:---:|
| **优惠券代码** | 公共“密码”(例如 `SUMMER20`)。 | **不可更改：** 请谨慎选择。 |
| **优惠券类型** | 现金返还选择 `结算券`，规则促销选择 `折扣券`。 | 必填 |
| **面值** | **[现金抵扣]** 设置确切的货币金额。直接影响损益表 (P&L)。 | 结算券必填 |
| **数量类型** | `固定` 用于制造稀缺性；`动态` 用于无限的病毒式传播活动。 | 必选 |
| **单个用户限制** | 防止单个用户重复使用或滥用。**(专业建议：对于高价值代码，请设置为 1)**。 | 建议设置 |
| **允许匿名** | 最适用于电子商务“访客结账”以提高转化率。 | 开关 |
| **领取周期** | “宣传窗口”——客户何时可以查看并领取代码。 | 战略范围 |
| **使用周期** | “核销窗口”——代码在结账时实际生效的时间。 | 战略范围 |

## 了解优惠券类型：何时、如何以及为什么？

并非所有促销活动都是一样的。该小程序提供了三种不同的**优惠券类型**，以处理不同的业务需求。选择正确的类型可确保应用正确的 UI 字段和结账逻辑。

### 1. 结算券 (Settlement Vouchers - “礼品卡”模式)
**为什么：** 当您想要发放具有**固定现金价值**且像货币一样运作的优惠券时，请使用此类型。
- **工作原理：** 您定义一个**面值金额** (Face Value Amount)（例如 $50）。兑换时，它会按该固定价值减少总发票金额。与折扣不同，这被记录为支付结算。
- **最适用于：** 礼品卡、针对不满客户的补偿券或预付商店信用积分。

### 2. 折扣券 (Discount Vouchers - “促销码”模式)
**为什么：** 用于传统的营销促销，其中折扣取决于**规则和条件**。
- **工作原理：** 您定义**处理方式** (Treatments)（例如 20% 折扣）和**应用规则**（例如最低消费 $100）。折扣是在购物车验证过程中计算的。
- **最适用于：** 节日促销、买一送一或网红推荐码。

### 3. 奖励券 (Reward Vouchers - “忠诚度”模式)
**为什么：** 用于生成**忠诚度积分**，而不是（或除了）降价。
- **工作原理：** 触发会员系统中积分的累积。
    - **客户积分 (Customer Points):** 奖励给最终用户的积分。
    - **零售商积分 (Retailer Points):** 奖励给商家/零售商的积分。
- **过期逻辑：** 您可以设置“客户积分过期天数”，以在您的忠诚度计划中创造紧迫感。
- **最适用于：** 会员里程碑奖励或活动出席积分。

## 快速入门指南

### 针对营销：创建促销活动 (折扣类型)

**目标：** 创建一个“黑五”促销码，满 RM 200 减 RM 50。

1. 导航至 **Voucher Management** 并点击 **“+” (新增)**。
2. 将 **Voucher Type** 设置为 `DISCOUNT`。
3. 将优惠券代码设置为 `BF2024`。
4. 在 **Apply Rules** 中，设置 `Minimum Spend` = `200`。
5. 在 **Treatment Value** 中，设置 `Discount Amount` = `-50`。
6. 将状态设置为 **Published**。

{{< figure src="/images/voucher-management-applet/voucher-edit.png" alt="优惠券创建表单" caption="创建活动：设置您的促销码并定义其活动状态" >}}

### 针对客户成功：发放结算券 (结算类型)

**目标：** 向特定客户发放 50 元的“抱歉”券，其功能类似于现金。

1. 在 Voucher Management 中点击 **“+” (新增)**。
2. 将 **Voucher Type** 设置为 `SETTLEMENT`。
3. 输入 **Face Value Amount** = `50`。
4. (可选) 在 **Validity Target** 中，搜索特定客户的电子邮件，以确保*只有他们*可以使用。
5. 将状态设置为 **Published**。

## 深度探索：高级配置

### 安全与隐私控制
- **Softpins:** 生成一个额外的秘密 PIN 码（6-30 个字符），高安全性优惠券兑换时需要提供。
- **Activation Pins:** 需要在销售点使用秘密 PIN 码“激活”优惠券后才能使用。
- **URL Keys:** 生成唯一的“一键应用”链接。用户点击链接时，优惠券会自动注入其会话/购物车。

### 序列号生成 (前缀逻辑)
在批量生成优惠券（例如 1,000 个代码）时，使用**前缀配置**来保持记录整齐。
- **示例：** 设置前缀为 `VIP-`，字符长度为 `8`。系统将生成类似于 `VIP-AX72J9K1` 的代码。

### 成功框架：9 个选项卡

结合这些层级来构建坚不可摧的促销活动。

| 框架层级 | 您实际控制的内容 |
|:---:|---|
| **1. 蓝图** | 身份标识、类型和全局发行限制。 |
| **2. 时机** | 领取窗口与兑换窗口。 |
| **3. 受众** | 谁受邀参加？ (VIP、新用户或所有人)。 |
| **4. 门槛规则** | “解锁”条件 (例如“消费满 RM100”)。 |
| **5. 预算上限** | 总财务敞口 (例如“满 500 次兑换后停止”)。 |
| **6. 商品目标** | 哪些商品符合条件？ (全店范围 vs 特定清仓商品)。 |
| **7. 奖励计算** | 具体的计算逻辑 (减免金额、百分比折扣或忠诚度积分)。 |
| **8. 例外情况** | 针对特殊边缘情况的进阶“If/Then”逻辑。 |
| **9. 地域范围** | POS/分店限制 (例如“仅限线上”或“仅限 Pavilion 分店”)。 |

## 常见问题解答

**问：优惠券创建后可以更改其类型吗？**  
答：不能。因为逻辑（面值 vs 处理规则）从根本上是不同的，您必须在开始时选择正确的类型。

**问：如果客户在总额低于优惠券面值的情况下使用结算券，会发生什么？**  
答：优惠券支付全额。根据您的 POS 配置，剩余余额可能会作废或保留供将来使用。

**问：奖励积分如何分配？**  
答：当兑换 **Reward** 类型优惠券时，积分会自动添加到 **Membership Admin Applet** 中的客户配置文件中。
